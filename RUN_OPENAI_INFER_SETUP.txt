run_openai_infer.sh 실행 가이드

중요 업데이트 (2026-03-24)
- `qwen_moe_host` 재현은 [QWEN_MOE_HOST_ENV_SETUP.md](/home/milab/cvpr2026cctv/team01/minseok/qwen_tv/QWEN_MOE_HOST_ENV_SETUP.md) 기준입니다.
- `./setup_qwen_moe_host.sh`는 이제 기본적으로 conda env file 기반 설치를 수행합니다.
- 아래 예시도 모두 `qwen_moe_host` 환경 기준으로 맞췄습니다.

개요
- 이 프로젝트의 `run_openai_infer.sh`는 다음 작업을 순서대로 수행합니다.
  1. `.env`를 읽습니다.
  2. `OPENAI_BASE_URL`에 연결되는 vLLM OpenAI 호환 서버가 아직 없으면 자동으로 실행합니다.
  3. `BASE_VIDEO_URL`에 연결되는 비디오용 HTTP 서버가 아직 없으면 자동으로 실행합니다.
  4. `inference_qwen35_openai.py`를 실행합니다.

사전 준비 사항
- `conda`가 설치된 Linux 머신
- 로컬에서 vLLM을 띄울 경우 NVIDIA GPU 환경
- PATH에 `curl`이 있어야 함
- 이 저장소가 로컬에 체크아웃되어 있어야 함
- 첫 실행 시 Hugging Face 모델 다운로드를 위한 인터넷 연결 필요

권장 디렉터리
- 저장소 루트를 다음처럼 가정합니다.
  `/path/to/qwen_tv`

1. conda 환경 만들기
- 최신 재현 파일은 아래 3개입니다.
  - main conda env file: `environment_qwen_moe_host.yml`
  - raw conda lock: `conda_explicit_qwen_moe_host.txt`
  - pinned pip list: `requirements_qwen_moe_host.txt`
- 다른 사람이 conda로 바로 환경을 만들게 하려면 `environment_qwen_moe_host.yml` 사용이 권장입니다.

명령어:

```bash
cd /path/to/qwen_tv
./setup_qwen_moe_host.sh
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate qwen_moe_host
```

이미 같은 이름의 환경이 있고 다시 만들고 싶다면:

```bash
cd /path/to/qwen_tv
RECREATE=1 ./setup_qwen_moe_host.sh
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate qwen_moe_host
```

메인 conda env file 대신 raw lock 경로를 쓰고 싶을 때는 아래처럼 실행합니다:

```bash
cd /path/to/qwen_tv
RECREATE=1 INSTALL_MODE=lock ./setup_qwen_moe_host.sh
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate qwen_moe_host
```

2. `.env` 준비하기
- `/path/to/qwen_tv/.env` 파일을 만듭니다.
- 예시는 아래와 같습니다.

```bash
HF_TOKEN=your_huggingface_token_if_needed
OPENAI_BASE_URL=http://localhost:8000/v1
OPENAI_API_KEY=EMPTY
MODEL_ID=Qwen/Qwen3.5-35B-A3B-FP8
BASE_VIDEO_URL=http://localhost:8002
DATASET_ROOT=/path/to/qwen_tv/dataset
LOG_DIR=/path/to/qwen_tv/logs
VLLM_TP_SIZE=2
KEEP_SERVERS=0
```

설명
- `HF_TOKEN`은 선택한 모델이 인증된 다운로드를 요구할 때만 필요합니다.
- 로컬 vLLM을 쓸 경우 `OPENAI_API_KEY=EMPTY`로 두어도 됩니다.
- `VLLM_TP_SIZE`는 실제로 보이게 할 GPU 개수와 맞추는 것이 좋습니다.
- `KEEP_SERVERS=1`로 두면 추론 종료 후에도 vLLM과 HTTP 서버를 유지합니다.

3. 특정 GPU에서 실행하기
- 예를 들어 여러 GPU가 있는 머신에서 GPU 6, 7만 사용하고 싶다면:

```bash
cd /path/to/qwen_tv
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate qwen_moe_host
CUDA_VISIBLE_DEVICES=6,7 ./run_openai_infer.sh \
  --video_path dataset/videos/-2UPLUV7JLg_00.mp4
```

중요
- 프로세스 내부에서는 `CUDA_VISIBLE_DEVICES`로 노출한 GPU만 존재합니다.
- 따라서 `CUDA_VISIBLE_DEVICES=6,7`로 실행하면 프로그램 내부에서는 이를 로컬 GPU 0, 1처럼 보게 됩니다.
- `run_openai_infer.sh`는 `--gpus` 인자를 받지 않습니다. GPU 선택은 `CUDA_VISIBLE_DEVICES`와 `VLLM_TP_SIZE`로 제어합니다.

8. 8000 또는 8002 포트가 이미 사용 중일 때
- `.env`에서 포트를 바꾸면 됩니다.

예시:

```bash
OPENAI_BASE_URL=http://localhost:8100/v1
BASE_VIDEO_URL=http://localhost:8102
```

그 다음은 평소처럼 실행하면 됩니다.

```bash
CUDA_VISIBLE_DEVICES=6,7 ./run_openai_infer.sh --video_path dataset/videos/-2UPLUV7JLg_00.mp4
```

9. 자주 발생하는 문제
- `vllm is required but not installed in the current environment.`
  conda 환경이 제대로 활성화되지 않았거나 환경 생성이 실패한 경우입니다.

- `OPENAI_BASE_URL is not local and is not reachable`
  설정한 vLLM endpoint가 잘못되었거나 서버가 내려가 있는 경우입니다.

- `base_video_url is not local and is not reachable`
  설정한 비디오 HTTP endpoint가 잘못되었거나 `DATASET_ROOT`가 잘못된 경우입니다.

- 모델 다운로드 또는 인증 실패
  `.env`에 올바른 `HF_TOKEN`을 넣거나 실행 전에 셸에서 export 해야 합니다.

- OOM 또는 매우 느린 초기화
  기본 모델이 크기 때문에 발생할 수 있습니다. 더 많은 GPU를 쓰거나, `VLLM_TP_SIZE`를 맞추거나, `--model_id`로 더 작은 모델을 선택해야 할 수 있습니다.

10. 최소 실행 예시

```bash

./setup_qwen_moe_host.sh
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate qwen_moe_host
printf '%s\n' \
  'OPENAI_BASE_URL=http://localhost:8000/v1' \
  'OPENAI_API_KEY=EMPTY' \
  'BASE_VIDEO_URL=http://localhost:8002' \
  "DATASET_ROOT=$(pwd)/dataset" \
  "LOG_DIR=$(pwd)/logs" \
  'VLLM_TP_SIZE=2' \
  > .env
CUDA_VISIBLE_DEVICES=6,7 ./run_openai_infer.sh --video_path dataset/videos/-2UPLUV7JLg_00.mp4
```
